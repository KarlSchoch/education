Positioning

This section will cover the five main ways for how to position elements using CSS within a webpage and demonstrate how these are used and what they look like.

Approach 1: Static

This is the default positioning for every element and is applied by using the position argument within a CSS element as illustrated below. Elements will appear within the normal "flow" of the HTML document.

.static-positioning {
            position: static;
        }

Approach 2: Relative

Relative positioning maintains elements within the "flow" of the HTML document, but allows you to shift them based on the left, right, top, and bottom properties. 
.relative-positioning {
            position: relative;
            left: 5%;
            width: 95%;
        }
Relative Positioning Example

This is an example of the above relative positioning styling applied to a paragraph, which results in it being "pushed in" from the left by 10%. Please note that we adjusted the width parameter to avoid "overflowing" the viewport.

Approach 3: Absolute

Absolute positioning removes documents from the normal document flow. Instead, you specify the distance that the element should be from it's ancestor's sides

This "ancestor" is also referred to as the positioning contexts and it's initial containing block. By default, the initial containing block is the initial viewport. You can change this default behavior by setting positioning on one of the element's ancestors

.absoltue-positioning {
            position: absolute;
            left: 20%;
            background-color: yellowgreen;
        }

Approach 4: Fixed

Fixed positioning is very straight forward: It ensures that an element stays at the same position regardless of scrolling. This is very useful for things like navigation headings, and the code below was used to create this page's navigation heading.

.fixed-positioning {
            position: fixed;
            top: 0;
            color: white;
            background-color: black;
        }

Approach 5: Sticky

This header is used so that you can make an element "stick" to a certain point on the screen after you scroll down by some amount. For instance, if you have an element that starts two thirds down the page but you want to always stay 50px from the top of the screen, you would use sticky positioning to achieve this effect.

You can achieve this through the following code, which we applied to this page to ensure that the various levels of headings provide users with conceptual "breadcrumbs" to show how what they are seeing on the screen fits into the larger context of technologies and concepts

h1 {
    position: sticky;
    top: 75px;
    margin: 2px 0em;
}
h2 {
    position: sticky;
    top: 114px;  /* offset calculation: top (75px) + h1 element (37) + first half h2 element margin (2) */
    margin: 2px 0em;
}
h3 {
    position: sticky;
    top: 143px;  /* previous + h2 element (27px) +  second half h2 element margin (2px) + first half h3 element margin (2px) */
    margin: 2px 0em;
}
h4 {
    position: sticky;
    top: 167px; /* previous + h3 element (22px) +  second half h3 element margin (2px) + first half h4 element margin (2px) */
    margin: 2px 0em;
}
